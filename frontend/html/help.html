// tips

smart query The LLM generates an optimised Elasticsearch DSL query before searching. Best for specific, structured questions. Disable for broad keyword searches or if the model is slow.
multi-query For complex questions (e.g. lateral movement), the LLM may generate multiple ES queries covering different event types and combine the results automatically.
concise Limits the response to 3–5 sentences. Useful for quick triage when you don't need full detail.
results Controls how many log records are retrieved and sent to the LLM. Higher values give more context but slow down the response. 20–50 is a good default.
indices Select only the indices relevant to your question. Querying fewer indices is faster and reduces noise. Use all only when you don't know which source to target.
deepseek-r1 Thinking models show their reasoning before answering. The reasoning block auto-collapses when the answer starts. Best for complex investigations where you want to see the logic.

// authentication & access

Show me all failed login attempts in the last 24 hours
Lists accounts with authentication failures — useful for spotting brute force or password spray activity.
Which accounts have the most login failures?
Aggregates failure counts by username. High counts on service accounts or admin accounts are high priority.
Are there accounts with successful logins after multiple failures?
Identifies potential successful brute force — failure bursts followed by a success from the same IP or account.
Show me logins outside of business hours
Surfaces access events between 18:00–07:00 or on weekends — common indicator of insider threat or compromised credentials.
Give me a list of usernames and how often they signed in
Frequency table of authentications per user. Unusually high or low counts may warrant investigation.
Which users logged in from more than one country?
Impossible travel — simultaneous or rapid logins from geographically distant locations.

// lateral movement

Are there signs of lateral movement?
Broad sweep — covers authentication across multiple hosts, remote service use, SMB/RDP connections, and pass-the-hash indicators. Maps to MITRE T1021, T1550.
Which users authenticated to multiple systems in a short timeframe?
Accounts accessing many hosts quickly may indicate automated lateral movement or credential reuse after compromise.
Show me remote desktop or SMB connections between internal hosts
Peer-to-peer RDP and SMB activity between workstations is unusual and often indicates lateral movement (T1021.001, T1021.002).
Are there any service account logins from workstations?
Service accounts authenticating interactively or from user workstations is a common sign of pass-the-ticket or stolen credentials.

// privilege escalation & persistence

Are there any new local administrator accounts created recently?
Newly created privileged accounts may indicate persistence (T1136) or privilege escalation (T1078).
Show me privilege escalation events
Looks for UAC bypasses, sudo usage, token impersonation, and group membership changes.
Which accounts were added to privileged groups?
Group membership changes to Domain Admins, Administrators, or sudo-capable groups are high-value events (T1098).
Are there scheduled tasks or services created by unusual accounts?
Persistence via scheduled tasks (T1053) or new services (T1543) created by non-admin accounts warrants investigation.

// process execution & malware

Show me PowerShell commands run by users
PowerShell abuse (T1059.001) is one of the most common living-off-the-land techniques. Look for encoded commands, download cradles, or unusual parent processes.
Are there processes spawned from unusual parent processes?
Word/Excel spawning cmd.exe or PowerShell is a classic malicious macro indicator. Helps identify T1566 phishing outcomes.
Find any processes running from temp or user-writable directories
Malware frequently executes from %TEMP%, AppData, or Downloads to avoid detection in system directories.
Are there signs of credential dumping?
Looks for LSASS access, Mimikatz-related activity, ntds.dit access, and SAM database reads (T1003).

// network & data exfiltration

Show me connections to external IP addresses
Outbound connections to non-RFC1918 addresses — useful for C2 detection (T1071) or data exfiltration (T1041).
What are the top destination ports and how many connections to each?
Unusual ports (4444, 8080, 1337, high ephemeral ports) may indicate C2 channels or non-standard services.
Are there signs of data exfiltration?
Looks for large outbound transfers, access to bulk sensitive files, cloud sync activity, and staging behaviour (T1560, T1048).
Find DNS queries to unusual or newly registered domains
DNS-based C2 (T1071.004) and DGA (Domain Generation Algorithm) traffic often appear as high-frequency queries to random-looking domains.

// investigation & triage

Summarise all activity from IP address 10.0.0.50
Full timeline of events associated with a specific source IP — useful when triaging an alert or investigating a known-bad indicator.
What did user jsmith do over the last 7 days?
Reconstructs a user's activity timeline — authentication, process execution, network connections, and file access.
Give me a timeline of events between 14:00 and 16:00 yesterday
Narrows the search window to a specific incident period — useful when you know when an alert fired.
Map these events to the MITRE ATT&CK framework
Asks the LLM to classify observed behaviours against ATT&CK techniques and provide tactic/technique IDs for reporting.
What are the most critical findings across all indices?
High-level triage sweep — identifies the most suspicious events across all available log sources.
